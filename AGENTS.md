# AGENTS.md

## 목적

이 저장소에서는 여러 에이전트가 병렬 또는 순차로 작업할 수 있다.  
작업 중복, 문맥 손실, 진행 상태 혼선을 줄이기 위해 공통 상태 문서를 사용한다.

새로 작업을 시작하는 모든 에이전트는 먼저 아래 문서를 읽고 현재 상태를 파악해야 한다.

- `PLAN.md`
- `PROGRESS.md`

## 문서 역할

### `PLAN.md`

`PLAN.md`는 "무엇을 해야 하는가"를 관리하는 문서다.

- 전체 목표
- 현재 스코프
- 큰 작업 단위
- 우선순위
- 다음에 해야 할 일
- 아직 시작하지 않은 작업
- 결정되었지만 구현되지 않은 항목

즉, `PLAN.md`는 앞으로의 작업 계획과 남은 일을 정리하는 문서다.

### `PROGRESS.md`

`PROGRESS.md`는 "어디까지 진행되었는가"를 관리하는 문서다.

- 완료된 작업
- 현재 진행 중인 작업
- 최근 변경 사항
- 작업 중 발견된 이슈
- 다음 에이전트가 바로 이어서 해야 할 일
- 참고할 구현 메모

즉, `PROGRESS.md`는 현재 진행 상황과 작업 이력을 정리하는 문서다.

## 작업 시작 규칙

새로 작업을 시작하는 에이전트는 반드시 다음 순서를 따른다.

1. `AGENTS.md`를 읽는다.
2. `PLAN.md`를 읽고 해야 할 작업과 우선순위를 파악한다.
3. `PROGRESS.md`를 읽고 현재까지의 진행 상태와 최근 변경 사항을 파악한다.
4. 그 다음 실제 분석, 구현, 수정 작업을 시작한다.

## 작업 중 문서 갱신 규칙

작업을 수행한 에이전트는 필요한 경우 아래 원칙에 따라 문서를 갱신한다.

- 계획이 바뀌면 `PLAN.md`를 갱신한다.
- 실제로 진행된 내용이 생기면 `PROGRESS.md`를 갱신한다.
- 완료되지 않은 작업은 `PLAN.md`에 남긴다.
- 완료된 작업과 현재 상태는 `PROGRESS.md`에 남긴다.

## 문서 사용 원칙

- `PLAN.md`와 `PROGRESS.md`는 항상 최신 상태를 유지한다.
- 문서만 읽어도 다음 에이전트가 바로 작업을 이어갈 수 있어야 한다.
- 큰 작업은 가능한 한 작은 단위로 나누어 기록한다.
- 구현 세부사항보다 "무엇이 끝났고 무엇이 남았는지"가 먼저 드러나야 한다.
- 문서 내용이 충돌하면 최신 실제 코드 상태에 맞게 정리한다.

## 현재 기준

현재 이 저장소에서 문서 역할은 다음과 같이 구분한다.

- `AGENTS.md`: 에이전트 작업 규칙과 문서 운영 규칙
- `PLAN.md`: 해야 할 일, 작업 계획, 우선순위
- `PROGRESS.md`: 현재 진행 상태, 완료 내역, 다음 작업 안내
- `docs/mvp-spec.md`: 프로젝트 MVP 요구사항 명세
- `docs/setup-guide.md`: 개발 셋업 및 기술 방향 가이드
