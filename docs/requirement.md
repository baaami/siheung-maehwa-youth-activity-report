# 역할

## 관리자 (청소년 지도사)

- 관리자는 /admin 페이지에서 동아리 별로 활동일지들을 관리할 수 있음

## 청소년

- 활동일지 작성과 활동 일지 작성한 것을 읽을 수 있음

# 제약 사항

- 영속성이 필요한 데이터들은 이후 DB 연결을 하므로 우선 파일에 저장한다.

# MVP 기준 변경 사항
- 제출 상태 관리는 하지 않습니다.

# 화면

## 청소년 관점

### 홈화면

- 상단에 "매아리 활동일지" 문구
- 1행에 2개 동아리 씩 2열로 동아리를 선택이 가능해야 한다.

### 동아리 등록 화면

- public/activity-report-pdf-format.png에 내용이 모두 매핑될 수 있도록 항목들을 청소년들이 입력할 수 있으면 됨
  - 현재 기준으로 /student/clubs/club-media/reports/new 화면을 참고하면됨
- 청소년들이 입력하는 항목들은 UX 친화적일 필요성이 존재
- public/activity-report-pdf-format.png 이미지가 청소년 지도사가 청소년이 등록한 리포트를 다운로드 했을 때 최종 결과물 PDF 형태이다.

- 활동 소감, 활동 내용은 200자 이상 작성해야 등록이 작성되어야 한다.

## 청소년 지도사 관점

### 홈화면

- /admin 페이지가 청소년 지도사 관리 홈 페이지이다.
- 상단에는 '매아리 활동일지 관리자 페이지'가 존재한다.
- 1행에 2개 동아리 씩 2열로 동아리를 선택이 가능해야 한다.

### 동아리 활동일지 리스트 화면

- 제출, 미제출 상태는 관리하지 않는다 존재하지 않아도 된다.
- 연도 별로 필터링하여 볼 수 있어야 한다.
- 활동일지들은 각각 일자, 활동명, 참여 인원, 상세 정보가 있으면되고 가장 우측에 선택할 수 있는 버튼이 필요하다.
- 활동일지 별로 상세 내용을 볼 수 있어야 한다.
- 활동일지들 여러 개를 선택해서 PDF 다운로드 받을 수 있어야 한다.

### 활동일지 상세 화면

- 동아리 활동일지 내용 수정이 가능해야한다.
- PDF 다운로드가 가능해야 한다.
- 활동 사진들은 다운로드 받을 수 있는 인터페이스가 있어야 한다.
