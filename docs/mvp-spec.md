# 동아리 활동일지 PDF 프로젝트 MVP 명세

## 1. 프로젝트 목표

학생이 동아리 활동일지를 등록하고, 관리자인 청소년지도사가 제출된 활동일지를 선택하여 HTML/CSS 기반 PDF로 변환한 뒤 하나의 PDF 파일로 다운로드할 수 있는 웹 서비스를 구축한다.

MVP 단계에서는 로그인과 보안 고도화는 제외하고, 역할 분리와 핵심 업무 흐름 구현에 집중한다.

## 2. 사용자 역할

- 학생
- 청소년지도사(관리자)

초기 화면에서 역할 버튼을 선택하여 진입한다.

## 3. 핵심 기능 범위

### 학생

- 여러 개의 동아리에 속할 수 있다.
- 본인이 속한 동아리 목록을 볼 수 있다.
- 동아리를 선택해 활동일지를 작성할 수 있다.
- 제출 전까지는 활동일지를 수정할 수 있다.
- 제출 후에는 조회만 가능하고 수정은 불가하다.
- 삭제 기능은 제공하지 않는다.

### 관리자

- 전체 동아리 목록을 조회할 수 있다.
- 동아리 추가/수정/삭제가 가능하다.
- 특정 동아리의 활동일지 목록을 조회할 수 있다.
- 활동일지 검색/필터링/수정이 가능하다.
- 제출된 활동일지를 여러 개 선택해 하나의 PDF 파일로 다운로드할 수 있다.
- PDF 생성 시 일부 내용이 1페이지를 초과해 잘린 경우, 관리자 화면에서 해당 사실을 확인할 수 있어야 한다.

## 4. 활동일지 데이터 규칙

### 기본 정보

- 동아리명
- 일자
- 활동명
- 활동내용
- 소감
- 사진 2장

### 참가자 정보

- 참가자 명단은 개별 참가자 단위로 입력한다.
- 각 참가자마다 다음 정보를 입력한다.
  - 이름
  - 성별
  - 연령대

### 연령대 구분

- 초등
- 중등
- 고등
- 후기 청소년(20~24)
- 청년(25~34)

### 인원 집계

- 참가자 입력값을 기준으로 성별/연령대별 인원을 자동 집계한다.
- 총 인원도 자동 계산한다.

## 5. 입력 제약 조건

- 활동내용: 공백 포함 최소 200자, 최대 300자
- 소감: 공백 포함 최소 200자, 최대 300자
- 사진: 정확히 2장 업로드
- 사진 크기 제한: 각 이미지당 최대 15MB
- 제출 가능 조건:
  - 필수 항목이 모두 입력되어야 한다.
  - 사진 2장이 모두 업로드되어야 한다.

## 6. 상태 규칙

- 활동일지 상태는 다음 두 가지로 관리한다.
  - 미제출
  - 제출
- 학생은 제출 전 상태에서만 수정 가능하다.
- 관리자는 제출 여부와 관계없이 수정 가능하다.
- 관리자가 수정해도 상태는 그대로 유지한다.
- 수정 이력 저장은 MVP 범위에서 제외한다.

## 7. 활동일지 구조

- 활동일지 1건 안에 하루의 여러 활동을 기록할 수 있다.
- 다만 출력 템플릿은 1페이지 내에 들어가야 하므로, 내용이 길 경우 PDF에서 잘릴 수 있다.
- 잘린 경우 서버 로그를 남기고, 관리자 화면에도 잘림 여부를 표시한다.

## 8. PDF 출력 규칙

- PDF 생성 방식은 HTML/CSS를 PDF로 변환하는 방식으로 구현한다.
- 제출된 활동일지만 PDF 다운로드 대상이 된다.
- 관리자가 여러 활동일지를 선택하면, 활동일지별로 PDF 1페이지씩 생성한다.
- 선택된 여러 페이지를 하나의 PDF 파일로 묶어 다운로드한다.
- 출력 순서는 사용자가 선택한 순서를 따르지 않아도 되며, 별도 순서 변경 기능은 제공하지 않는다.
- 파일명 규칙은 다음 형식을 따른다.
  - `{동아리명}_{YYYYMMDD}_활동일지묶음.pdf`

## 9. 출력 템플릿

```md
- 동아리명: ( )
- 일자: YYYY년 MM월 DD일
- 참가자 명단:
- 인원: 총 ( )명
  | 구분 | 초등 | 중등 | 고등 | 후기 | 청년 |
  |:---:|:---:|:---:|:---:|:---:|:---:|
  | 남 | 명 | 명 | 명 | 명 | 명 |
  | 여 | 명 | 명 | 명 | 명 | 명 |
- 활동명:
- 활동내용:
  -
  -
- 소감:
- 사진: [사진 1 삽입 위치] / [사진 2 삽입 위치]
```

## 10. 템플릿 작성 규칙

- 총 인원과 성별/연령대별 인원은 참가자 명단을 기준으로 자동 계산한다.
- 활동내용은 개조식으로 요약해 출력한다.
- 소감은 자연스러운 문장으로 정리한다.
- 사진은 업로드된 2장을 템플릿 하단에 배치한다.
- 정보가 누락된 경우 `(정보 필요)`로 표시한다.

## 11. 화면 구성 초안

### 공통

- 초기 진입 화면
  - 학생 버튼
  - 관리자 버튼

### 학생 화면

- 내가 속한 동아리 목록
- 동아리별 활동일지 목록
- 활동일지 작성/수정 화면
- 제출 후 읽기 전용 상세 화면

### 관리자 화면

- 동아리 목록
- 동아리 추가/수정/삭제
- 동아리별 활동일지 목록
- 검색/필터
- 활동일지 상세/수정
- 다중 선택 PDF 다운로드
- PDF 잘림 여부 표시

## 12. MVP에서 제외하는 항목

- 로그인/회원가입
- 권한 인증 고도화
- 수정 이력 관리
- 제출 승인/반려 프로세스
- 업로드 사진 3장 이상 지원
- PDF 페이지 수 조정 기능
- PDF 순서 직접 정렬 기능

## 13. MVP 이후 확장 후보

- 실제 인증/인가 도입
- 관리자 수정 이력 저장
- PDF 미리보기
- PDF 생성 작업 큐 분리
- 이미지 자동 압축/리사이즈 고도화
- 제출 상태 외 승인 상태 추가
- 정렬 및 다운로드 옵션 확장
